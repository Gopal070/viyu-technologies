# Viyu Didi Deployment Guide (Render)

This guide provides step-by-step instructions for deploying the **Viyu Didi** application (Frontend + Backend) to Render.

---

## 1. Push Code to GitHub

Since Render deploys directly from your repository, you first need to push your local code to GitHub.

1.  **Initialize Git**:
    ```powershell
    git init
    git add .
    git commit -m "Initial commit for deployment"
    ```
2.  **Create a New Repository** on [GitHub](https://github.com/new).
3.  **Link and Push**:
    ```powershell
    git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
    git branch -M main
    git push -u origin main
    ```

---

## 2. Deploy to Render via Blueprint

The `render.yaml` file in the root directory automates the setup.

1.  Log in to the [Render Dashboard](https://dashboard.render.com).
2.  Click **New +** and select **Blueprint**.
3.  Connect your GitHub repository.
4.  Render will detect the `render.yaml` file and show the **web service** (viyu-technologies).
5.  Click **Apply**.

---

## 3. Configure Environment Variables

Some variables are sensitive and must be added manually in the Render dashboard after the first (failed) deployment build starts.

Go to: **Dashboard > viyu-technologies > Environment**.

| Key | Value | Notes |
| :--- | :--- | :--- |
| `MONGO_URI` | `mongodb+srv://...` | Your MongoDB Atlas connection string. |
| `EMAIL_USER` | `your-email@gmail.com` | Used for contact form notifications. |
| `EMAIL_PASS` | `your-app-password` | Gmail App Password (not your login password). |
| `ADMIN_EMAIL` | `recipient@email.com` | Where form enquiries will be sent. |

> [!TIP]
> `JWT_SECRET` is automatically generated by the Blueprint, but you can override it if needed.

---

## 4. Run Initial Database Seed

To populate your live database with initial products, services, and the admin user:

1.  Go to the **Shell** tab of your service in the Render Dashboard.
2.  Run the following command:
    ```bash
    cd backend && npm run seed
    ```

---

## 5. Maintenance & Updates

- **Automatic Deploys**: Every time you `git push` to the `main` branch, Render will automatically rebuild and redeploy your site.
- **Node Version**: The project is configured to use Node.js 20.
- **Frontend Files**: Any changes to `.html` files in the root will be served instantly after deployment.
